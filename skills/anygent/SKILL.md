# Anygent — 나만의 Claude 에이전트 만들기

> 4단계 인터뷰를 통해 맞춤형 CLAUDE.md를 생성합니다.

## 사용법

```
/anygent                          # 인터뷰 시작
/anygent 마케팅 콘텐츠 작성 에이전트   # 설명과 함께 시작
```

---

## 실행 지침

사용자가 `/anygent`를 실행하면 아래 4단계를 순서대로 진행하세요.

---

### STEP 1 — PLAN 🗺️

사용자에게 다음 질문들을 한꺼번에 물어보세요 (숫자 목록 형태로):

```
Anygent에 오신 걸 환영합니다! 🤖
나만의 Claude 에이전트를 만들어드릴게요.

몇 가지 질문에 답해주세요:

1. 이 에이전트의 주요 목적은 무엇인가요?
   예) SNS 콘텐츠 작성, 고객 문의 응답, 코드 리뷰

2. 주로 누가 사용하나요?
   예) 마케팅 팀, 나 혼자, CS 담당자

3. 어떤 상황에서 에이전트를 호출하나요?
   예) 포스팅 작성이 필요할 때, 이메일 초안이 필요할 때

4. 에이전트가 절대 하면 안 되는 것이 있나요?
   예) 경쟁사 언급, 법적 조언, 가격 협상

5. 답변 스타일은 어떻게 원하나요?
   예) 친근하고 캐주얼, 프로페셔널, 간결하게
```

사용자의 답변을 받으면 STEP 2로 진행하세요.

---

### STEP 2 — RESEARCH 🔍

수집한 정보를 바탕으로 내부적으로 분석하고 사용자에게 요약을 보여주세요:

```
분석 완료! 다음과 같이 이해했습니다:

🎯 에이전트 이름: [도출한 이름]
📋 핵심 역할: [요약]
👤 사용 대상: [대상]
⚡ 트리거 상황: [상황]
🚫 금지사항: [목록]
🗣️ 톤앤매너: [스타일]

이 방향으로 진행할까요? (수정사항이 있으면 말씀해 주세요)
```

사용자가 확인하면 STEP 3으로 진행.

---

### STEP 3 — DO ⚙️

현재 디렉토리에 `CLAUDE.md` 파일을 생성하세요. 아래 형식을 따르되 사용자의 답변으로 채워주세요:

```markdown
# [에이전트 이름]

## 역할
[핵심 역할 설명 — 1~2문장]

당신은 [역할] 전문가입니다. 다음 역할을 수행합니다:

## 핵심 역할
- [능력 1]
- [능력 2]
- [능력 3]
(사용자 답변 기반으로 3~5개 생성)

## 언제 호출하나요?
[트리거 상황 설명]

## 작업 방식
- [작업 방식 1]
- [작업 방식 2]
- [작업 방식 3]
(사용자 목적에 맞는 베스트 프랙티스 제안)

## 출력 형식
[사용자가 원하는 결과물 형태 설명]

## 금지사항
[사용자가 언급한 금지사항 목록]

## 톤앤매너
[스타일 설명]

## 예시 프롬프트
- "[실제로 쓸 법한 요청 예시 1]"
- "[실제로 쓸 법한 요청 예시 2]"
- "[실제로 쓸 법한 요청 예시 3]"
```

파일 생성 후 STEP 4로 진행.

---

### STEP 4 — CHECK ✅

생성된 CLAUDE.md를 기반으로 샘플 시나리오 3개를 직접 실행해보세요:

```
CLAUDE.md 생성 완료! 이제 에이전트를 테스트해볼게요.

샘플 시나리오 3가지를 시뮬레이션합니다:

**테스트 1**: [현실적인 요청]
→ [에이전트가 어떻게 응답할지 짧게 시뮬레이션]

**테스트 2**: [다른 유형의 요청]
→ [시뮬레이션]

**테스트 3**: [엣지 케이스]
→ [시뮬레이션]

---
✅ 에이전트가 준비됐습니다!

📁 CLAUDE.md가 현재 폴더에 저장됐어요.
🚀 이제 claude 명령어를 실행하면 이 에이전트로 시작합니다.

수정이 필요하면 언제든지 /anygent를 다시 실행하거나 CLAUDE.md를 직접 편집하세요.
```
